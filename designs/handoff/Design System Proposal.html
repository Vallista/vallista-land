03 — Token architecture

한 번 더 정리하는 토큰.

현재는 Primitive와 Semantic이 섞여 있어 “이 색을 어디서 가져와야 하지?”가 문서 없이는 불분명합니다. Vercel Geist · Radix · Primer처럼 3계층 토큰으로 재정렬하면, Primitive 수정이 Semantic을 통해 컴포넌트까지 자연스럽게 흐릅니다.

L1 · PRIMITIVE
Raw values

색·수치·문자열의 원본. UI에서 직접 참조 금지.

gray.50gray.100gray.200 gray.900terra.500mint.500
L2 · SEMANTIC
Intent mapping

“배경 · 본문 · 강조 · 위험” 등 의도를 Primitive에 매핑. UI는 이 레이어만 사용.

surface.canvas
→ gray.50
surface.raised
→ gray.100
text.primary
→ gray.900
accent.emphasis
→ terra.500
status.success
→ green.500
L3 · COMPONENT
Component-scoped

컴포넌트가 자기만의 토큰을 가짐. 리스킨·테마 주입이 쉬워집니다.

button.primary.bg → accent.emphasis
button.primary.fg → text.onAccent
button.primary.hover.bg → accent.emphasisHover
button.primary.focus.ring→ focus.ring
card.bg → surface.raised
card.border → border.subtle
PRIMITIVE → SEMANTIC → COMPONENT  ·  수정은 왼쪽에서, 소비는 오른쪽에서

@vallista/design-system 07 · Motion · Depth
06 — Motion & depth

움직임에도 작법이 있습니다.

현재는 전역 transition: *가 모든 속성에 0.15s를 먹입니다. 목적별 easing + duration 토큰을 정의하고, 전역 와일드카드는 해제합니다.

Easing tokens
ease.standard (0.2, 0.8, 0.2, 1)

대부분의 UI 전환 — 호버, 포커스, 토글.

ease.entrance (0, 0, 0.2, 1)

등장 — 모달, Toast, 드로어 열림.

ease.exit (0.4, 0, 1, 1)

사라짐 — 닫힘, dismiss.

Duration
fast · 120ms base · 200ms slow · 320ms expressive · 520ms
Elevation tokens
elevation.0 — flat inline
elevation.1 — resting card
elevation.2 — raised dropdown · hover
elevation.3 — overlay popover · toast
elevation.4 — modal dialog · command

다크모드에서는 그림자 대신 border + bg shift로 깊이 표현.

Focus & a11y
focus.ring

모든 interactive 컴포넌트에 :focus-visible 전용 링 토큰을 적용. 마우스 포커스는 유지하지 않고 키보드에서만 표시.

reduced motion
@media (prefers-reduced-motion){
  --duration-fast: 0ms;
  --duration-base: 0ms;
  /* ease 유지, 기간만 0 */
}
APCA Lc 60+ keyboard-first aria-live

왜 vanilla-extract으로
옮겼는가

런타임 CSS-in-JS는 편했지만, 모노레포의 빌드가 무거워질수록 초기 로드가 눈에 띄게 느려졌다. SSR에서 styled-components의 serverStyleSheet가 토큰 변경과 겹치면 FOUC가 나기도 했다.

theme.css.ts Copy
import { createGlobalTheme } from '@vanilla-extract/css';
export const vars = createGlobalTheme(':root', {
  colors: { surface: '#fafafa' }
});

위처럼 타입 추론이 되는 토큰을 컴포넌트에서 소비한다. 하이라이트는 accent-soft로.

10 — Roadmap

5주 안에, 파괴 없이.

모든 변경은 기존 컴포넌트의 Public API를 유지합니다. Phase 1~2에서 토큰과 타이포그래피를 정비하고, 3~4에서 블로그 경험을 갱신합니다.

WEEK 1
Phase 1 · Tokens
  • ✓ Primitive / Semantic / Component 3계층 정의
  • ✓ OKLCH 색상 팔레트 · neutral 10단계
  • ✓ gray / success / warning 토큰 교정
  • ✓ focus.ring · elevation · easing 토큰 추가
WEEK 2
Phase 2 · Type & motion
  • ✓ Inter · Fraunces · Plex Mono self-host
  • ✓ 8종 타입 스케일 + Heading 재정렬
  • ✓ 전역 transition 제거 · motion token 적용
  • ✓ prefers-reduced-motion 대응
WEEK 3
Phase 3 · Components
  • ○ Button · Input · Tag · Modal 토큰 적용
  • ○ Storybook 스토리 & 접근성 체크
  • ○ Skeleton · EmptyState · Callout 신규
  • ○ CodeBlock 컴포넌트 (복사 · 하이라이트 · 라인 번호)
WEEK 4-5
Phase 4 · Blog
  • ○ Article 레이아웃 · TOC · 읽기 진행률
  • ○ Home 피드 재구성 · Series 카드
  • ○ 검색 (⌘K) · Tag cloud · Related
  • ○ SEO · OG 이미지 템플릿 표준화
