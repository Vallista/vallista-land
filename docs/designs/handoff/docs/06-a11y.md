# Accessibility — WCAG 2.1 AA

Not optional. Every route must pass. The rules below are the bar.

## Contrast (tested)

All combinations verified at 16px body size.

| Foreground / Background             | Ratio  | AA body | AA UI |
|-------------------------------------|--------|---------|-------|
| `--ink` on `--bg`                   | 17.4:1 | ✓       | ✓     |
| `--ink-2` on `--bg`                 | 9.4:1  | ✓       | ✓     |
| `--ink-3` on `--bg`                 | 5.3:1  | ✓       | ✓     |
| `--ink-4` on `--bg`                 | 3.1:1  | ✗ body  | ✓ large/decorative only |
| `--accent` (blue-500) on `--bg`     | 5.1:1  | ✓       | ✓     |
| `--ink-2` on `--bg-soft`            | 8.9:1  | ✓       | ✓     |
| `--accent-hover` on `--accent-tint` | 7.2:1  | ✓       | ✓     |

**Rule:** `--ink-4` is for **decorative only** — placeholders, disabled labels, meta that has a text alternative. Never body copy.

## Focus

```css
*:focus-visible {
  outline: none;
  box-shadow: var(--focus-ring);
}
```

- Every interactive element must have a visible `:focus-visible` state.
- Focus ring uses a 2px offset (bg color) + 2px accent at 55% opacity. This works on both light + dark.
- Never `outline: none` without replacing the ring. Never `:focus` alone — always `:focus-visible`.

## Keyboard

- Tab order must match DOM order. No `tabindex` > 0.
- Modal (search palette, mobile menu) opens → focus moves to first input. Closes → focus returns to the trigger.
- Palette: ↑/↓ navigates, ↵ opens, `Esc` closes. Results are `role="option"` inside `role="listbox"`.
- Skip link: first focusable element on every page. Hidden until focused.

```html
<a href="#main" class="skip-link">본문으로 바로 가기</a>
```

```css
.skip-link {
  position: absolute;
  left: -9999px;
}
.skip-link:focus {
  left: 16px; top: 16px;
  z-index: var(--z-modal);
  padding: 10px 16px;
  background: var(--ink);
  color: #fff;
  border-radius: var(--radius-3);
}
```

## Semantics

- **Heading hierarchy strict.** h1 → h2 → h3. Never skip levels. Use the type scale (not heading levels) for visual size.
- **Landmarks** on every page: `<header>`, `<nav>`, `<main>`, `<footer>`.
- **aria-current="page"** on the active nav rail item — drives the style, not a CSS class.
- **Lists** for list-shaped content (nav items, article rows). Not `<div>` spam.
- **Buttons vs links**: button triggers an action on this page, link navigates. Don't use one for the other.

## Images

No raster images in v1 — covers are inline SVG patterns (`docs/05-blog-spec.md § Covers`). When we add real images:

- Astro `<Image>` component, required `alt` attribute.
- `alt=""` only for purely decorative images (and rarely).
- Cover SVGs: `role="img"` + `<title>` child with the pattern name.

## Motion

See `docs/07-motion.md § 6`. `prefers-reduced-motion: reduce` collapses all durations to 0.01ms. Required. No exceptions.

## Color independence

Status must never be communicated with color alone.

| State       | Color      | Non-color signal           |
|-------------|------------|----------------------------|
| active      | green      | dot + label "active"       |
| maintenance | neutral    | dot + label "maintenance"  |
| archived    | amber      | dot + label "archived"     |

Chips use `.chip.dot::before { content:""; }` to add the dot — it's part of the component, not ad-hoc.

## Dark mode

- Contrast ratios re-verified in dark mode. All of the above still pass.
- Toggle is a real button with `aria-pressed` reflecting state.
- `prefers-color-scheme: dark` sets the default. `localStorage.theme` overrides.

```html
<button aria-pressed="false" aria-label="다크 모드 전환" data-theme-toggle>
  <!-- icon -->
</button>
```

## Forms

- Every input has a visible `<label>` (not just placeholder).
- Error states use `aria-invalid="true"` + `aria-describedby` pointing to the error message.
- Newsletter form: client-side validates, server responds within 2s or shows a "전송 중…" state.

## Reading

- Article body: `line-height: 1.85`. Below 1.6 is too tight for Hangul at this density.
- Never justify Korean text (`text-align: justify` forbidden).
- Never `word-break: break-all`. Default is `keep-all`, globally.

## Testing

Before merging any page:

1. Keyboard-only walkthrough — can you reach every interactive element and dismiss every overlay?
2. Screen reader sanity: VoiceOver or NVDA reads landmark → heading → first link in a sensible order.
3. Zoom to 200%. Nothing clipped, nothing horizontally scrolls (except code blocks).
4. Run axe-core against a production build:

```sh
pnpm dlx @axe-core/cli dist/ --exit
```

Zero violations required. Warnings are OK but justify in PR.
